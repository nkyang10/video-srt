# SenseVoiceSmall 日本語 ASR アーティファクト対応

SenseVoiceSmall は日本語音声に対して広東語/中国語の文字を誤出力する既知の現象があります。これは ASR モデルが日本語と中国語（広東語を含む）の音響特徴を混同することに起因します。

加えて、固有名詞（地名・料理名・キャラ名・魚名）の認識精度が低く、同一単語が複数の異なる形で誤認識される傾向があります。

## カテゴリ別アーティファクトパターン

## コンテキスト別の注意点

### テック系（チュートリアル・レビュー）
- のだ/なのだ 調が頻出 → 文体の特徴として保持
- モデル名（Claude, GPT, Gemini, Qwen, Llama, Ollama）は固有名詞として保持
- 英単語カタカナ崩れ（エアイ→エーアイ, CLI, MIT等）の修正が多数

### 旅行・観光系（Vlog・旅番組）
- 地名の表記揺れが激しい → 同一地名が3〜5種の誤認識形で出る前提で修正
- 施設名・温泉名の固有名詞は推測に頼る部分あり → 前後文脈から判断
- 食品名（明太子→明太粉など）の誤認識が頻出
- です/ます調が基本、口語的表現（めちゃくちゃ、っていう）あり

### 水族館・動物系
- 魚介類の名前が特に不安定（真鯛→マ鯛/話鯛/話題を）
- タカアシガニ、アナゴ、サメなどが漢字混同で全く別の単語に

## 対策

LLM 正規化フェーズで対応可能。ポイント:

1. **コンテキストから推測** — 前後の字幕ブロックから話者の意図を復元。地名の場合は特に有効
2. **同一動画内での一貫性** — 同じ固有名詞（沼津、真鯛など）は動画内で全て統一
3. **のだ/なのだ スタイル** — チュートリアル系話者に頻出。文体の特徴として保持すべき
4. **相槌の「嗯」→「うん」** — 最も頻出パターン。確実に変換
5. **モデル名の誤認識** — Claude→クロード, GPT→GPT, Gemini→Gemini, Qwen→Qwen, Llama→Llama
6. **英単語のカタカナ崩れ** — エアイ→エーアイ, シーエルアイ→CLI, エムアイテー→MIT 等
7. **食品名の一貫性** — 明太粉は全て明太子に。あんかけパッサ→パスタ

## 検出方法

処理前に以下のパターンで grep 確認:

```bash
grep -cP '[\x{4e00-\x{9fff}]' input.srt  # 中国語文字の含有率
grep -cE '(マズ|ズ工|ま豆|豆工)' input.srt  # 日本語地名崩れ
```

日本語動画で中国語文字が 5% 以上含まれる場合、アーティファクトが発生している可能性が高い。
